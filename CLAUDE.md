# Claude Code 项目说明

## 项目概述

这是一个**微信文章新闻分析项目**，用于分析订阅的微信公众号文章（主要是数据中心、算力、AI相关），自动提取新闻并生成日报。

## 当用户说"分析新闻"时该如何处理

**重要**：这是你需要注意的关键事项！

当用户说"分析昨天18点到今天18点的新闻"或类似需求时：

1. **不要直接假设使用本项目**
   - 第一反应应该是使用 `WebSearch` 从网络搜索
   - 因为用户可能有多种意图

2. **正确的处理流程**：
   ```
   如果用户首次提出"分析新闻"需求：
   ┌─────────────────────────────────┐
   │ 先问用户：是用网络搜索，       │
   │ 还是使用这个微信文章分析项目？ │
   └─────────────────────────────────┘
   ```

3. **只有在以下情况下才使用本项目**：
   - 用户明确说"用这个项目"、"用那个脚本"
   - 或者对话已经在本项目的上下文中
   - 用户提到的日期范围符合微信文章发布周期

### 日报格式（简化版）

```
# 算力、数据中心、AI动态日报 | 2026-MM-DD

## 🔥 今日重点（AI筛选）
1. [新闻标题]
   [链接]
   [摘要]
   **为什么重要**：AI分析的重要性
   **这意味着**：AI分析的影响

## 💰 融资与投资
## 🏢 数据中心建设
## 🔧 技术动态
## ⚠️ 风险
## 📊 其他
```

**核心特点**：
- AI智能筛选今日重点（最多3条），不是按金额，而是综合行业影响、里程碑事件、战略意义等维度
- AI自动分类新闻
- 每条新闻保留链接和摘要

### 重要修复（历史记录）
- **时间字段问题**：API返回两个时间字段
  - `create_time` ：文章创建时间（不准确）
  - `sent_info.time`：实际发布时间（✓ 正确）
- 代码已修复使用 `sent_info.time` 进行过滤
- **报告截断问题**：如果报告被截断，重新运行 `daily-analysis-llm.js` 和 `optimize-report.js`

## 其他说明

- LLM配置：使用 yovole/DeepSeek-V3.2
- 11个公众号关注领域：数据中心、算力、云计算、AI、大数据
- AI智能去重：使用 LLM 进行语义去重
- Agent智能优化：基于 report-optimizer-agent.md 配置，自动识别"有料"新闻并生成深度解读
- 报告格式：Markdown，采用新版"有温度、有观点、互动式"格式（详见下文「新版日报格式模板」）

### Agent优化规则（report-optimizer-agent.md）

**"有料"标准**：
- 百亿级以上的重大投资（如 100亿美元、200亿元）
- 行业转折点事件（如首次、突破、挑战、垄断）
- 重大政策/监管变化（如管制、禁令、制裁）
- 颠覆性技术突破（如首条、首创、里程碑）
- 重大并购/上市动态

**优化逻辑**：
- 如有"有料"新闻：精简开头的"今日要闻分析"（2-3句），保留所有新闻，在文末添加"🔍 深度解读"
- 如无"有料"新闻：保持原样，仅作格式优化

## 新版日报格式模板

从2026年3月10日起，日报采用新版"有温度"格式，增加三个增加器：
- **情绪增加器**：每条头条都有"吐槽/点赞/预警"
- **知识增加器**：每天一个"今日新词"（技术术语/行业概念）
- **社交增加器**：每天末尾加互动问题

### 格式核心变化说明

| 变化点 | 目的 | 手动/AI |
|--------|------|----------|
| 今日一句话 | 用口语化替代官方通稿式开篇 | 手动 |
| 今日新词 | 每天积累一个知识点，建立个人知识库 | 手动/可脚本化 |
| 我的看法（头条后） | 每条重要新闻给明确态度，引发讨论 | 手动 |
| 今日盘点表格 | 信息密度更高，便于快速扫读 | AI可自动归类 |
| 三个槽点 | 释放情绪，不只是"看到了"而是"有感觉" | 手动 |
| 今天的互动 | 给读者回复的理由 | 手动 |
| 引流提示 | 为"5个铁粉"目标做准备 | 每期固定 |

### 如何使用

- **新日报生成**：运行 `daily-analysis-llm.js` 后，再用 `optimize-report.js` 优化，即可按新版格式输出
- **历史日报重写**：如需将历史日报改写为新格式，可手动按模板编辑，或使用 LLM 辅助转换
