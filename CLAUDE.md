# Claude Code 项目说明

## 项目概述

这是一个**微信文章新闻分析项目**，用于分析订阅的微信公众号文章（主要是数据中心、算力、AI相关），自动提取新闻并生成日报。

## 当用户说"分析新闻"时该如何处理

**重要**：这是你需要注意的关键事项！

当用户说"分析昨天18点到今天18点的新闻"或类似需求时：

1. **不要直接假设使用本项目**
   - 第一反应应该是使用 `WebSearch` 从网络搜索
   - 因为用户可能有多种意图

2. **正确的处理流程**：
   ```
   如果用户首次提出"分析新闻"需求：
   ┌─────────────────────────────────┐
   │ 先问用户：是用网络搜索，       │
   │ 还是使用这个微信文章分析项目？ │
   └─────────────────────────────────┘
   ```

3. **只有在以下情况下才使用本项目**：
   - 用户明确说"用这个项目"、"用那个脚本"
   - 或者对话已经在本项目的上下文中
   - 用户提到的日期范围符合微信文章发布周期

## 如何使用本项目

### 前置条件
1. 确保 `wechat-article-exporter` 服务正在运行（端口 3000）
2. 项目位置：`D:\业余兴趣\wechat-article-exporter`
3. 启动命令：`npm run dev`

### 运行日报分析
```bash
cd "D:\业余兴趣\wechat-article-skill"
node daily-analysis-llm.js
```

### 时间范围说明
- 脚本分析的是：**昨天18:00 到今天18:00** 的微信公众号文章
- 来源公众号：卓乎、WMedia、企业网D1net、DTDATA、全球技术地图、数智猿、雷峰网、DeepKnowledge、AI芯界、艾笛网、IDC圈
- 输出位置：`D:\业余兴趣\新闻分析结果\算力、数据中心、AI动态日报{mm}.md`

### 重要修复（历史记录）
- **时间字段问题**：API返回两个时间字段
  - `create_time` ：文章创建时间（不准确）
  - `sent_info.time`：实际发布时间（✓ 正确）
- 代码已修复使用 `sent_info.time` 进行过滤
